# VNOP SOP Hub — 고도화 기획 (Enhancement Roadmap)

현재 버전은 **검색 가능한 정적 프로토타입**입니다. 18개 SOP를 4개 언어(영어 기준 + 한/중/일)와
다크모드로 제공하며, 자격 증명은 안전하게 마스킹되어 어디든 배포할 수 있습니다.

아래는 실제 운영에서 가치가 큰 순서대로 정리한 고도화 단계입니다. 각 항목은 독립적으로
진행 가능하며, 프로토타입의 "무빌드 / 무의존성" 장점을 최대한 유지하는 방향으로 설계했습니다.

---

## 🟢 Phase 1 — 콘텐츠 완성도 (현재 구조에서 바로 가능)

| # | 항목 | 설명 | 난이도 |
|---|------|------|--------|
| 1 | **본문 다국어 번역** | 현재 제목·요약만 번역됨. `blocks`를 `{ en:[…], ko:[…] }` 구조로 확장해 본문 전체를 한/중/일로 제공. `planning-plugin:translate` 또는 번역 검수 워크플로 활용. | 중 |
| 2 | **용어집(Glossary)** | HCN, ELLIS, VCC, AGT, DMC, Override 등 약어를 호버 툴팁/별도 페이지로 제공. 신규 입사자 온보딩 효과. | 하 |
| 3 | **SOP 간 상호 링크** | 본문에서 "SOP-008" 언급 시 클릭하면 해당 SOP로 이동 (이미 `related` 필드 존재 → 자동 링크화). | 하 |
| 4 | **"최근 본 SOP" / 즐겨찾기** | localStorage 기반. 자주 쓰는 SOP를 상단 고정. | 하 |
| 5 | **변경 이력(Changelog)** | 각 SOP의 version/revised 기반으로 "최근 개정" 배지 및 변경 내역 표시. | 중 |

## ⚠️ 보안 관련 권고 (즉시 검토)

현재 자격 증명은 마스킹되어 있으나, 원문 SOP에는 다음과 같은 민감 정보가 포함되어 있었습니다:

- 호텔/포털 로그인 비밀번호, Access Key, 법인카드(AMEX) 정보
- 직원 이메일·계정 ID
- 내부 비즈니스 규칙(예: Agoda 거절 사유 처리, Override 수수료 티어)

**권고 사항:**
1. **실제 자격 증명은 비밀번호 관리자(1Password, Bitwarden 등)** 에만 보관하고 문서에는 절대 평문으로 기재하지 않습니다.
2. 이 사이트는 **비공개 저장소 / 인증된 호스팅** 으로 운영합니다.
3. `source-docs/` 폴더의 **원본 .docx 파일에는 평문 자격 증명이 그대로 남아 있으므로**, 저장소에서 제외(`.gitignore`)되어 있습니다. 이 폴더는 절대 커밋하지 마세요.

> 원본 docx는 `source-docs/`에 보관하며 `.gitignore`로 제외됩니다. (앱 데이터 폴더 `data/`와 이름이 겹치지 않도록 분리했습니다.)

## 권장 진행 순서 (요약)

```
지금          →  Phase 1-(1) 본문 번역, (2) 용어집, (3) SOP 상호링크
1–2주 내      →  Phase 2-(6) 의사결정 트리, (7) 체크리스트 실행 모드
배포 정착     →  Phase 3-(11) PR 검수 워크플로, (13) 접근 제어
장기          →  Phase 4-(16) SOP 챗봇(RAG)
```

각 단계는 현재 코드베이스(`data/sops.js` 콘텐츠 모델, `assets/app.js` 렌더러)를 그대로
확장하는 형태로 설계되어, 한 번에 하나씩 안전하게 도입할 수 있습니다.
