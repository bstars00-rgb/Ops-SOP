## 1. 개요 / 목적
VNOP 운영팀이 18개 표준업무절차서(SOP-001~018)를 **언제든 검색·열람하고 현장 업무에 바로 활용**할 수 있도록 만든 다국어·로그인 기반 정적 웹 애플리케이션. 영어를 기준으로 한국어·중국어·일본어·베트남어를 지원하며, 다크모드·반응형을 갖추고 GitHub Pages로 배포된다.

## 2. 범위
**포함:** SOP 열람/검색/필터, 5개 언어, 다크모드, 반응형(노트북·태블릿·모바일), 로그인+콘텐츠 암호화, 용어집, 의사결정 트리(가이드 모드), 인터랙티브 체크리스트, 즐겨찾기/최근 본 SOP, 번역 검수 배지, 자동화 테스트.
**제외(보류):** SOP 챗봇(RAG) — 정적 사이트에 API 키를 안전히 둘 수 없어 별도 프록시 서버가 필요, 현시점 보류. 사용 분석/백엔드, 실기기 크로스브라우저 자동화(현 E2E는 Chromium 기준).

## 3. 사용자 & 주요 시나리오
- **운영 담당자:** 상황 발생 시 SOP를 검색하거나 카테고리/즐겨찾기로 찾아 절차를 확인. 분기형 상황은 가이드 모드로 정확한 조치 도출. 일일 점검은 체크리스트로 진행.
- **신규 입사자:** 용어집과 본문 용어 툴팁으로 약어(HCN, ELLIS 등) 학습.
- **다국어 사용자:** 우측 상단에서 언어 전환(영/한/중/일/베).

## 4. 기능 명세

### 4.1 인증 & 보안
- **로그인 게이트:** 사이트 진입 시 ID+비밀번호 요구. 둘을 합쳐 AES 키를 유도하므로 ID·비밀번호 모두 맞아야 복호화.
- **콘텐츠 암호화:** SOP 데이터는 AES-256-GCM 암호문(`data/sops.enc.js`)으로만 게시. 비밀번호 없이는 파일을 받아도 내용 열람 불가(PBKDF2-SHA256, 200k iterations).
- **세션 유지:** 같은 탭 세션 동안 자동 잠금 해제(sessionStorage), 탭 종료 시 재로그인.
- **자격증명 마스킹:** 원문 SOP의 비밀번호·Access Key·법인카드는 `🔒 [Stored in the team password manager]` 플레이스홀더로 대체.
- **CSP:** 엄격한 Content-Security-Policy(`script-src 'self'`) 적용.
- **원본 보호:** 평문 마스터(`source-docs/`)는 gitignore로 미공개.

### 4.2 다국어 (i18n)
- 지원 언어: en(기준), ko, zh, ja, vi. UI 문자열·카테고리·SOP 제목·요약·목적·**본문 전체** 번역.
- 언어별 폴백: 누락 시 영어로 자동 폴백.
- `purpose`/`body`는 언어 맵 `{en,ko,zh,ja,vi}` 구조.

### 4.3 검색 / 필터 / 탐색
- 전문 검색: 제목·요약·목적·본문·태그·SOP 번호(전 언어 인덱싱), 결과 하이라이트.
- 카테고리 필터: 예약 처리·에이전트 요청·수동 예약·정산 및 분쟁 + ★즐겨찾기.
- 딥링크: SOP별 고유 URL(`#/sop/SOP-008`), 링크 복사.
- 최근 본 SOP 스트립, 즐겨찾기(localStorage).

### 4.4 SOP 상세
- 블록 기반 렌더링: 제목(h)·문단(para)·목록(list)·표(table)·콜아웃(info/warn/critical/note/tip)·키값(kv)·체크리스트(checklist).
- 사이드 목차(데스크톱), 메타(버전/시행일/개정일/작성/적용대상/관련 SOP), 인쇄.
- **본문 용어 툴팁:** 약어(HCN, ELLIS, AGT 등)에 탭/키보드로 정의 팝오버.
- **검수/임시번역 배지:** 비영어 언어에서 미검수 시 "⚠ 임시 번역" 표시. `reviewed.{lang}=true` 설정 시 "✓ 검수 완료"로 전환.

### 4.5 용어집 (Glossary)
- 19개 용어(약어+정식명칭+설명), 5개 언어, 검색 가능한 모달.

### 4.6 가이드 모드 (의사결정 트리)
- 분기형 12개 SOP(001·002·003·004·005·006·007·008·009·010·016·017)에 예/아니오 인터랙티브 플로우.
- 단계 표시·뒤로·처음부터·최종 조치 카드, 5개 언어, aria-live 지원.

### 4.7 인터랙티브 체크리스트
- 체크리스트 블록(예: SOP-015, SOP-017)을 체크 가능, 진행률(x/n) 표시, localStorage에 SOP별 저장.

### 4.8 표시/UX
- 다크/라이트 테마(시스템 감지+기억), 반응형(폰 1열·태블릿 2열·데스크톱 다열), 터치 타깃 확대, 키보드 접근성·모달 포커스 트랩.

## 5. 데이터 모델
마스터: `source-docs/sops.json` = `{ secret, categories[], glossary[], trees{}, data[] }` (gitignore, 평문).
```
SOP = {
  id, cat, version, effective, revised, by, applicable?, related?, reviewed?,
  title:{en,ko,zh,ja,vi}, summary:{…}, purpose:{…},
  tags:[…], body:{ en:[block], ko:[block], … }
}
block.t ∈ h | para | list | table | callout | kv | checklist
```

## 6. 아키텍처 / 파일 구조
- 무빌드 정적 웹앱(순수 HTML/CSS/JS). 데이터는 `<script>`로 로드(file:// 호환), 복호화는 Web Crypto.
- `index.html` · `assets/{app,auth,i18n,theme-init}.js` · `assets/styles.css` · `data/sops.enc.js`
- 부팅 흐름: `auth.js`가 복호화 성공 후 `window.SOP_*` 설정 → `window.bootSOPApp()` 호출.

## 7. 번역 / 빌드 / 테스트 / 배포 파이프라인
- 콘텐츠 수정: `source-docs/sops.json`(+`extras/<lang>.json`) 편집 → 병합 스크립트(`tools/merge*.js`) → `node tools/encrypt.js` → `data/sops.enc.js` 커밋·푸시.
- 테스트:
  - `npm test` — 데이터 무결성(zero-dep, 약 3,800 assert): 18 SOP×5언어, 블록 구조 일치, 용어집/트리 정합성, 핵심규칙(Agoda 'Fully Booked'·11:15·티어), 암호문 비밀유출 검사.
  - `npm run e2e` — Playwright E2E(로그인·검색·언어전환·트리·용어집), 자격증명은 env.
- 배포: GitHub Pages(main/root), `.nojekyll`.
